title: Automating Automation

Automation used to feel pretty simple to me.

You find a repetitive task, write a script, save some time, and move on. That was how I thought about it for a long time. Automation was mainly a tool for removing boring work. Useful, of course, but still pretty local. You automate one thing here, one thing there, and your day becomes slightly better.

But over time, I started to feel that automation is much bigger than that.

The real value is not just that it saves time. It changes what people spend their attention on. At first, we automate small steps. Then we automate a workflow. Then we improve the environment around that workflow, so people can make fewer mistakes and recover faster. Better defaults, better tooling, better systems. It is not only about doing the same work faster. It is also about changing the shape of the work itself.

Now AI is pushing this one step further.

It is not only helping people execute automated steps. It is also helping people build them. It can write draft code, suggest structure, and handle a lot of the glue work that used to consume so much time. So in a way, we are entering a stage where automation is starting to automate the creation of automation.

It seems this is the natural direction.

That does not mean engineers become less important. I would say it changes what becomes important. The value is less about manually typing every detail, and more about deciding what should exist, defining the right constraints, and checking whether the result is actually correct. The bottleneck moves upward, from execution to judgment.

So when I think about AI, I do not mainly think about replacement. I think about leverage. The distance between an idea and a working system is getting shorter. If that keeps improving, then more and more energy can be spent on choosing the right problems and designing the right systems, instead of manually stitching every piece together.

To me, this is where automation was always heading. Not machines replacing people, but machines helping people build better systems, faster. And if that is true, then engineering becomes more strategic, not less.
